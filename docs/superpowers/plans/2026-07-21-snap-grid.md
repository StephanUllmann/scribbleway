# Snap / Grid / Alignment Guides Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Shift-constrained drawing/resizing (45° lines/arrows, square/circle rects/ellipses) and live alignment snap guides when dragging or resizing shapes against other shapes' edges and centers.

**Architecture:** Interaction-only feature — no new shape `QVariantMap` keys and no Excalidraw schema changes. Shift-angle/aspect math lives in QML at the draw and resize call sites. Alignment snap is computed in `OverlayController` (testable C++), applied inside `dragSelected` and via `Q_INVOKABLE` helpers that resize handlers call. Ephemeral guide lines are exposed as a `QVariantList` Q_PROPERTY and rendered in `main.qml`.

**Tech Stack:** Qt 6, C++20, QML.

## Global Constraints

* **No model keys:** Do not add snap/grid fields to shapes, `ShapeRoles`, D-Bus selection state, or Excalidraw convert paths.
* **Snap threshold:** `6.0` CSS-like pixels (constant `kSnapThreshold` in `OverlayController`).
* **Guides are ephemeral:** Cleared on pointer release / `endEdit()` / draw finalize-or-abort. Never persisted.
* **Shift semantics (no conflict with multi-select):**
  * **Draw** (`drawMouseArea`): Shift → angle/aspect constrain.
  * **Resize / endpoint drag** (`BaseShape.qml`): Shift → aspect 1:1 or 45° angle.
  * **Move drag** (`dragSelected`): Shift is *not* used for constrain; multi-select already uses Shift on press only.
* **What snaps:** Left / centerX / right and top / centerY / bottom of the moving geometry against the same six lines of every *non-moving* unlocked shape. Multi-selection move snaps the **union bounding box** of the selection as one rigid group.
* **Out of scope:** Visible background grid, configurable grid size, magnetic snap-to-grid, applet toggle UI (keep `snapEnabled` internal/default-on; optional Q_PROPERTY only if trivial).

```cpp
QRectF OverlayController::shapeBounds(const QVariantMap &shape) const
{
    const QString type = shape.value(QStringLiteral("type")).toString();
    if (type == QStringLiteral("rectangle") || type == QStringLiteral("ellipse")
        || type == QStringLiteral("text")) {
        return QRectF(shape.value(QStringLiteral("x")).toDouble(),
                      shape.value(QStringLiteral("y")).toDouble(),
                      shape.value(QStringLiteral("width")).toDouble(),
                      shape.value(QStringLiteral("height")).toDouble()).normalized();
    }
    if (type == QStringLiteral("line") || type == QStringLiteral("arrow")) {
        const QPointF a(shape.value(QStringLiteral("fromX")).toDouble(),
                        shape.value(QStringLiteral("fromY")).toDouble());
        const QPointF b(shape.value(QStringLiteral("toX")).toDouble(),
                        shape.value(QStringLiteral("toY")).toDouble());
        return QRectF(a, b).normalized();
    }
    if (type == QStringLiteral("freehand")) {
        const QVariantList points = shape.value(QStringLiteral("points")).toList();
        if (points.isEmpty())
            return {};
        QRectF r;
        bool init = false;
        for (const QVariant &pv : points) {
            QPointF p;
            if (pv.canConvert<QPointF>())
                p = pv.toPointF();
            else if (pv.typeId() == QMetaType::QVariantMap) {
                const QVariantMap pm = pv.toMap();
                p = QPointF(pm.value(QStringLiteral("x")).toDouble(),
                            pm.value(QStringLiteral("y")).toDouble());
            } else {
                continue;
            }
            if (!init) {
                r = QRectF(p, QSizeF(0, 0));
                init = true;
            } else {
                r = r.united(QRectF(p, QSizeF(0, 0)));
            }
        }
        return r;
    }
    return {};
}
```

```cpp
void OverlayController::collectSnapTargets(const QSet<int> &exclude,
                                           QVector<double> *xs,
                                           QVector<double> *ys) const
{
    xs->clear();
    ys->clear();
    const auto &shapes = m_shapesModel.shapes();
    for (int i = 0; i < shapes.size(); ++i) {
        if (exclude.contains(i))
            continue;
        if (shapes[i].value(QStringLiteral("locked")).toBool())
            continue; // still snap *to* locked shapes — only skip if you decide otherwise.
                      // Spec: snap TO all non-moving shapes including locked. Do NOT skip locked here.
        const QRectF b = shapeBounds(shapes[i]);
        if (!b.isValid())
            continue;
        xs->append(b.left());
        xs->append(b.center().x());
        xs->append(b.right());
        ys->append(b.top());
        ys->append(b.center().y());
        ys->append(b.bottom());
    }
}

// IMPORTANT: locked shapes ARE valid snap targets. Only exclude moving indices.
// Remove the locked continue above — final code must NOT skip locked targets.

double OverlayController::snapScalar(double value, const QVector<double> &targets,
                                     double threshold, double *matchedTarget)
{
    double bestDist = threshold;
    bool hit = false;
    double best = value;
    double bestT = value;
    for (double t : targets) {
        const double d = std::abs(value - t);
        if (d <= bestDist) {
            bestDist = d;
            best = t;
            bestT = t;
            hit = true;
        }
    }
    if (matchedTarget)
        *matchedTarget = hit ? bestT : std::numeric_limits<double>::quiet_NaN();
    return hit ? best : value;
}
```

Snap rule when multiple edges of the moving box can snap: evaluate left, center, right independently; pick the candidate with the **smallest absolute delta**; apply that single `dx` (same for y). This keeps the box rigid.

```cpp
// Pseudo for rigid rect snap (used by snapRect and dragSelected group box):
// candidatesX = [
//   targets.snap(left) - left,
//   targets.snap(cx)   - cx,
//   targets.snap(right)- right
// ]
// pick dx with min |dx| among those that actually matched (finite matchedTarget)
// same for dy
// emit one guide per axis that matched
```

```cpp
QVariantMap OverlayController::snapRect(int movingIndex, double x, double y, double w, double h)
{
    QSet<int> exclude;
    if (movingIndex >= 0)
        exclude.insert(movingIndex);
    // Also exclude any currently selected shapes if this index is part of multi-select resize
    // (resize is single-shape only today — exclude just movingIndex).

    QVector<double> xs, ys;
    collectSnapTargets(exclude, &xs, &ys);

    QRectF box(x, y, w, h);
    // normalize carefully: preserve sign of w/h for resize math by working on normalized copy for snap,
    // then re-apply. Simpler path: callers always pass positive w/h (BaseShape already clamps minSize).

    const double left = x;
    const double right = x + w;
    const double cx = x + w / 2.0;
    const double top = y;
    const double bottom = y + h;
    const double cy = y + h / 2.0;

    double ml = 0, mc = 0, mr = 0, mt = 0, mcy = 0, mb = 0;
    const double sl = snapScalar(left, xs, kSnapThreshold, &ml);
    const double sc = snapScalar(cx, xs, kSnapThreshold, &mc);
    const double sr = snapScalar(right, xs, kSnapThreshold, &mr);
    // deltas if hit (NaN matched => miss)
    struct Cand { double delta; double guide; bool hit; };
    auto cand = [](double snapped, double original, double matched) -> Cand {
        const bool hit = !std::isnan(matched);
        return { snapped - original, matched, hit };
    };
    const Cand cl = cand(sl, left, ml);
    const Cand cc = cand(sc, cx, mc);
    const Cand cr = cand(sr, right, mr);

    Cand bestX{0, 0, false};
    for (const Cand &c : {cl, cc, cr}) {
        if (!c.hit) continue;
        if (!bestX.hit || std::abs(c.delta) < std::abs(bestX.delta))
            bestX = c;
    }

    // identical for Y with top/cy/bottom ...

    QVariantList guides;
    if (bestX.hit)
        guides.append(QVariantMap{{QStringLiteral("orientation"), QStringLiteral("v")},
                                  {QStringLiteral("position"), bestX.guide}});
    // ... bestY ...

    setActiveSnapGuides(guides);

    QVariantMap out;
    out.insert(QStringLiteral("x"), x + (bestX.hit ? bestX.delta : 0.0));
    out.insert(QStringLiteral("y"), y + (bestY.hit ? bestY.delta : 0.0));
    out.insert(QStringLiteral("width"), w);
    out.insert(QStringLiteral("height"), h);
    return out;
}
```

- [ ] **Step 5: Integrate snap into `dragSelected`**

Current implementation: `overlaycontroller.cpp` ~1438–1478. After computing the proposed translation `(dx, dy)` from drag start, **before** applying to each shape:

1. Build union bounds of all `m_dragStartShapes` via `shapeBounds`.
2. Translate union by `(dx, dy)` → proposed group box.
3. `exclude` = all keys in `m_dragStartShapes`.
4. Run the same rigid-box snap as `snapRect` on the proposed group box → `(snapDx, snapDy)` adjustment relative to proposed, plus guides.
5. Effective translation = `(dx + snapDx, dy + snapDy)`.
6. Apply effective translation to every start shape (existing per-type branches).

```cpp
void OverlayController::dragSelected(double dx, double dy)
{
    if (m_dragStartShapes.isEmpty())
        return;

    // Union of drag-start bounds
    QRectF group;
    bool init = false;
    QSet<int> exclude;
    for (auto it = m_dragStartShapes.begin(); it != m_dragStartShapes.end(); ++it) {
        exclude.insert(it.key());
        const QRectF b = shapeBounds(it.value());
        if (!b.isValid())
            continue;
        group = init ? group.united(b) : b;
        init = true;
    }

    double adjDx = dx;
    double adjDy = dy;
    if (init) {
        const QRectF proposed = group.translated(dx, dy);
        // reuse internal snap on proposed → returns delta adjustment + sets guides
        // e.g. auto [sx, sy] = snapBoxDelta(proposed, exclude);
        adjDx = dx + sx;
        adjDy = dy + sy;
    }

    m_shapesModel.beginEdit();
    // ... existing loop using adjDx/adjDy instead of dx/dy ...
    m_shapesModel.endEdit();
}
```

```qml
function constrainLineDelta(dx, dy) {
    const len = Math.sqrt(dx * dx + dy * dy);
    if (len < 1e-6)
        return Qt.point(dx, dy);
    const angle = Math.atan2(dy, dx);
    const step = Math.PI / 4;
    const snapped = Math.round(angle / step) * step;
    return Qt.point(len * Math.cos(snapped), len * Math.sin(snapped));
}

// Anchor at drawStartPoint; mouse at (mx, my). Returns {x,y,w,h} for square/circle.
function constrainRectToSquare(sx, sy, mx, my) {
    let dx = mx - sx;
    let dy = my - sy;
    const ax = Math.abs(dx);
    const ay = Math.abs(dy);
    const side = Math.max(ax, ay);
    const sdx = (dx === 0 ? (mx >= sx ? 1 : -1) : Math.sign(dx)) * side;
    const sdy = (dy === 0 ? (my >= sy ? 1 : -1) : Math.sign(dy)) * side;
    return {
        x: Math.min(sx, sx + sdx),
        y: Math.min(sy, sy + sdy),
        w: Math.abs(sdx),
        h: Math.abs(sdy)
    };
}
```

## Design Notes (Implementer Reference)

### Snap priority
Per axis, among {left, center, right} (or {top, center, bottom}), choose the match with smallest `|delta|` within `kSnapThreshold`. At most **one vertical and one horizontal** guide at a time.

### Why C++ for snap
* `dragSelected` already owns multi-shape translation and `m_dragStartShapes`.
* Bounds extraction for freehand/line/rect already exists in `selectShapesInRect` — consolidate rather than reimplement in QML.
* Unit-testable without a QML scene.

### Why QML for Shift-draw
* Draw preview state (`previewW/H`, `drawStartPoint`) is entirely QML-local today (`main.qml` ~262–284).
* No C++ hook on draw move; adding one would be pure churn.

### Optional follow-ups (not this plan)
* Applet toggle for snap on/off (`snapEnabled` Q_PROPERTY + D-Bus).
* Visible background grid + snap-to-grid.
* Distance callouts between snapped edges.
* Snap during live draw to existing shapes (currently draw only has Shift-constrain).

## Acceptance Criteria (Summary)

1. **Shift constrain (draw):** lines/arrows → 45° multiples; rects/ellipses → square/circle; finalized geometry matches preview.
2. **Shift constrain (edit):** corner resize → 1:1; line endpoints → 45° about the fixed end.
3. **Alignment snap (move/resize):** edges and centers of moving geometry snap to non-moving shapes within 6px; rose `v`/`h` guides render at matched positions.
4. **Guides ephemeral:** empty after `endEdit` / release.
5. **No schema churn:** zero new `QVariantMap` keys, roles, D-Bus signals (aside from local QML-facing `activeSnapGuidesChanged` on the controller — not forwarded to the applet).
6. **Tests cover** snap hit, snap miss, guide clear, ignore-self exclusion.
